## Problem

Affiliate links generated by the automation pipeline were using the wrong Amazon Associates tag, causing all click-through revenue to be lost silently. No error is thrown — links still work, they just don't credit the account.

## Root Cause

`.env` contained a typo in the associate tag:

```
AMAZON_ASSOCIATE_TAG=goldenhomep0a-20  # wrong: 'a' instead of '6'
```

Correct value:
```
AMAZON_ASSOCIATE_TAG=goldenhomep06-20  # correct
```

The character `0a` (zero + letter a) was written instead of `06` (zero + six). Amazon links with wrong tags generate no commission.

## Fix Applied

Corrected `.env` line 16:
```
AMAZON_ASSOCIATE_TAG=goldenhomep06-20
```

Note: `github_daily_poster.py` had the correct default (`goldenhomep06-20`) hardcoded, so GitHub Actions posting was unaffected. Only the local `.env`-dependent scripts were impacted.

## Prevention

- Cross-reference the associate tag against the Amazon Associates dashboard URL on setup
- Add a startup check in pipeline scripts: validate tag format matches `[a-z0-9]+-[0-9]+`
- The tag is also stored correctly in GitHub Secrets — use that as the source of truth

## Audit Trail

Tag confirmed correct in:
- `github_daily_poster.py` default argument ✓
- `BUSINESS_BRAIN.md` reference ✓
- GitHub Actions secret AMAZON_ASSOCIATE_TAG ✓
- `.env` — was wrong, now fixed ✓
